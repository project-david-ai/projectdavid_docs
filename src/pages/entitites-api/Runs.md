A Run is a state-aware envelope that orchestrates one end-to-end interaction between an assistant and a thread.
The Run object serves multiple purposes in:
- Execution state
- Lifecycle tracker
- Control surface
- Telemetry & billing

---

### Create Run

### Request body
|                                                                                                                                                                                                                                                                                                                                      |
|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **`thread_id`**   `string`                                                                                                                                                                                                                                                                                                                             
The unique id of the thread associated with the run (`thread_L8PGee5yMiyl0TSVtjTxDC`).                                                                                                                                                                                                                                                                                  |
| **`assistant_id`** `string`                                                                                                                                                                                                                                                                                                                   
 The unique id of the assistant associated with the run                                                                                                                                                                                                                                                                            |
| **`instructions`** `string` `null` `Optional`                                                                                                                                                                                                                                                                                                                       
 A list of files attached to the message. This can be used to pass files to tools.                                                                                                                                                                                                                                                                                                                      |
| **`meta_data`** `<object>` or `null` `Optional`                                                                                                                                                                                                                                                                                                         
 A set of value pairs that can be attached to your run object. This is useful for storing additional arbitrary information about the assistant in a  structured format. These values can be used in queries via the API.Keys are strings with a maximum length of 64 characters. Values are strings with a maximum length of 512 characters.                                                                                                                                                                                                                                                                                                   |
 
The role assigned to the user or AI entity that has authored the message. 
 - `user` A message authored at the consumer level                                                                                                                                                                                                                                        
 - `assistant` A message generated by the assistant; usually in response to the most recent message from the user or tool.
 - `tool` A special case role used to inject output from tool calls into the dialogue; not available for consumer use.

print(run.model_dump_json())

### The run object
|                                                                                                                                                                                                                                                                                                                                       |
|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **`id`**   `string`                                                                                                                                                                                                                                                                                                                   
 Unique thread identifier (`run_eVRzeHoCyEl3SCMUcYx9xG`).                                                                                                                                                                                                                                                                              |
| **`user_id`**   `string`                                                                                                                                                                                                                                                                                                              
 The unique id of the user associated with the run (`user_gzZTWbTvB2XMfitDMeOEC4`).                                                                                                                                                                                                                                                    |
| **`cancelled_at`** `integer` or `null`                                                                                                                                                                                                                                                                                                
 Epoch timestamp (in seconds) for when the run was cancelled.                                                                                                                                                                                                                                                                          |
| **`completed_at`** `integer` or `null`                                                                                                                                                                                                                                                                                                
 Epoch timestamp (in seconds) for when the run was completed.                                                                                                                                                                                                                                                                          |
| **`created_at`** `integer` or `null`                                                                                                                                                                                                                                                                                                  
 Epoch timestamp (in seconds) for when the run was created.                                                                                                                                                                                                                                                                            |
| **`expires_at`** `integer` or `null`                                                                                                                                                                                                                                                                                                  
 Epoch timestamp (in seconds) for when the run expires.                                                                                                                                                                                                                                                                                |
| **`failed_at`** `integer` or `null`                                                                                                                                                                                                                                                                                                   
 Epoch timestamp (in seconds) for when the run failed.                                                                                                                                                                                                                                                                                 |
| **`instructions`**   `string`                                                                                                                                                                                                                                                                                                         
 System instructions to be used by the assistant for this run.                                                                                                                                                                                                                                                                         |
| **`last_error`**   `string`                                                                                                                                                                                                                                                                                                           
 An error that may be associated with the run, one of: `server_error`, `rate_limit_exceeded`, or `invalid_prompt`                                                                                                                                                                                                                      |
| **`max_completion_tokens`** `integer` or `null`                                                                                                                                                                                                                                                                                       
 The maximum number of tokens to be used in the course of this run— input/output inclusive.                                                                                                                                                                                                                                            |
| **`metadata`**   `<object>` or `null` `Optional`                                                                                                                                                                                                                                                                                      
 A set of value pairs that can be attached to your run object. This is useful for storing additional arbitrary information about the run in a  structured format. These values can be used in queries via the API.Keys are strings with a maximum length of 64 characters. Values are strings with a maximum length of 512 characters. |                                                                                                                                                                                                                                      |
| **`model`**   `string` or `null`                                                                                                                                                                                                                                                                                                      
 The model used for this run                                                                                                                                                                                                                                                                                                           |             
| **`object`**   `string`                                                                                                                                                                                                                                                                                                               |
 The object type                                                                                                                                                                                                                                                                                                                       |                        
| **`parallel_tool_calls`**   `boolean`                                                                                                                                                                                                                                                                                                 |                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          
 Whether to enable parallel function calling during tool use.                                                                                                                                                                                                                                                                          |
| **`required_action`**   `object` or `null`                                                                                                                                                                                                                                                                                            |                                                                                                                                                                                                                                                                                                                             
 Action required before continuing the run—usually tool call name with the input parameters.                                                                                                                                                                                                                                           |
| **`response_format`**   `string`                                                                                                                                                                                                                                                                                                      
 Specifies the format that the model must output. Compatible with. Some models will output responses in a structured format, if instructed to do so. For example, a setting to Setting to { "type": "json_schema", "json_schema": {...} } will enable output in a specific JSON schema.                                                |
| **`started_at`** `integer` or `null`                                                                                                                                                                                                                                                                                                  |                                                                                                                                                                                                                                                                                                             
 Epoch timestamp (in seconds) for when the run was started.                                                                                                                                                                                                                                                                            |                                                                                                                                                                                                                                                                                  |
| **`status`** `string` or `null`                                                                                                                                                                                                                                                                                                       
 The status of the response: `failed`, `in_progress`, `cancelled`, `queued`, or `incomplete`.                                                                                                                                                                                                                                          |                                                                                                                                                                                                                                                                                  |
| **`thread_id`** `string`                                                                                                                                                                                                                                                                                                              
 The unique id of the thread associated with the run (`thread_L8PGee5yMiyl0TSVtjTxDC`).                                                                                                                                                                                                                                                |
| **`thread_id`** `string`                                                                                                                                                                                                                                                                                                              
 The unique id of the thread associated with the run (`thread_L8PGee5yMiyl0TSVtjTxDC`).                                                                                                                                                                                                                                                |
| **`tool_choice`** `string` or `null`                                                                                                                                                                                                                                                                                                  
 Defines how the model should decide which tool (or combination of tools) to call when producing a response. Refer to the tools parameter for details on specifying the set of tools available to the model.                                                                                                                           |
| **`tools`** `array`                                                                                                                                                                                                                                                                                                                   | 
An array" of tools that the model may call while generating a response. To direct the model to a specific tool, use the `tool_choice` parameter. Tools fall into two categories:                                                                                                                                                      | 
| **Platform tools**: Provided natively (e.g. web search, file search) to extend the model’s capabilities.                                                                                                                                                                                                                              |<br>- **Consumer function calls**: User-defined functions with strongly typed arguments and outputs, allowing the model to invoke your own code. |
| **Consumer function calls**: User-defined functions with strongly typed arguments and outputs, allowing the model to invoke your own code.                                                                                                                                                                                            
|`truncation_strategy`— `auto` or `disabled`                                                                                                                                                                                                                                                                                                 
If set to auto, the thread will be truncated to the most recent `n` number of messages as the thread reaches the models maximum context length.| 
|`temperature` `string` or `null`                                                                                                                                                                                                                                                                                                 
The sampling temperature used for this run. If not set, defaults to 1.|
|`top_p` `string` or `null`                                                                                                                                                                                                                                                                                                 
Nucleus-sampling parameter. The model considers the top n% tokens when making its probabilistic determination. So 0.1 means that the top 10% will be considered. Use this or temperature to affect the models creativity, but not both. Some models may not support this parameter; in which any setting will be substituted with temperature.  |
|`tool_resources` `string` or `null` |                                                                                                                                                                                                                                                                                                
Per-tool resource map keyed by tool type. Example:`{"file_search":{"vector_store_ids":["vs_docs"]}}`                                                                                                                                                                                                                                        |
